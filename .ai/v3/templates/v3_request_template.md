# v3_request_template.md - Шаблон V3 Request

Версия: 0.2
Назначение: шаблон для Codex или человека перед отправкой V3-запроса внешнему чату.
Используй совместно с: [`create_v3_request_prompt.md`](../prompts/create_v3_request_prompt.md).

---

## Как использовать

1. Скопируй шаблон ниже.
2. Заполни все поля `[placeholder]`.
3. Используй commit-pinned GitHub raw links как default context mode.
4. Не требуй ручной передачи context files, если fallback не объявлен явно.
5. Явно укажи `current_stage`, чтобы не смешать package generation и import-stage.

---

```text
## V3 ID
[V3-YYYYMMDD-HHMMSS-<slug>]

## Task Title
[Короткое название задачи]

## Generated by
[codex | human | codex+human]

## Action
create

## Scope
[docs_only | workflow_docs | schemas | scripts | product_code]

## Current Stage
[external_artifact_generation_only | pre_kilo_package_revision | import_stage]

## Контекст задачи
[Краткое описание задачи + commit-pinned GitHub raw links]

## Описание задачи
[Что именно нужно создать]

## Allowed Paths
```
[список путей]
```

## Forbidden Paths
```
[список путей]
```

## Expected Files
```
[список путей]
```

## Package Format
zip

## Acceptance Criteria
```
[список критериев]
```

## Known Risks
```
[список рисков]
```

## No Repo Access Statement
Подтверди, что ты понимаешь:
- у тебя нет доступа к репозиторию;
- ты создаёшь ZIP-пакет, а не пишешь файлы напрямую;
- package может быть отдельно review-нут до import-stage;
- import возможен только позже, если он будет явно выбран.
```

## Подсказка

Если `Current Stage = external_artifact_generation_only`, request не должен делать вид, что Kilo import начинается немедленно.
