# v3_request_template.md — Шаблон V3 Request

Версия: 0.1 (Phase 3)
Назначение: шаблон для Codex или человека, заполняемый перед отправкой V3-запроса внешнему чату.
Используй совместно с: [`create_v3_request_prompt.md`](../prompts/create_v3_request_prompt.md).

---

## Как использовать

1. Скопируй шаблон ниже.
2. Заполни все поля, помеченные `[placeholder]`.
3. Поля с `<!-- comment -->` содержат подсказки по заполнению.
4. Передай заполненный шаблон внешнему чату вместе с project-specific контекстом.

## Подсказки по заполнению

### `v3_id`
- Всегда формат `V3-YYYYMMDD-HHMMSS-<slug>`.
- Slug — короткий идентификатор латиницей через дефисы: `v3-contracts`, `setup-guide`, `phase3-prompts`.
- Дата и время — UTC на момент создания запроса.

### `scope`
- Выбирай минимально необходимый scope. Не бери `workflow_docs`, если достаточно `docs_only`.
- `product_code` не используется в MVP (до Phase 8).

### `allowed_paths`
- Префикс `.ai/v3/` покрывает всю вложенность `.ai/v3/`.
- Указывай только то, куда реально будут писаться файлы.
- Сверяй со `scope`: `docs_only` + `src/` — невалидно.

### `forbidden_paths`
- Минимальный набор уже в шаблоне. Добавляй только project-specific запреты.
- `.env`, `secrets/`, `credentials/` — всегда в forbidden, даже если не в стандартном списке.

### `expected_files`
- Конкретные project-relative пути.
- Если файлов много, можно сгруппировать по директориям.
- Пример: `.ai/v3/prompts/create_v3_request_prompt.md`.
