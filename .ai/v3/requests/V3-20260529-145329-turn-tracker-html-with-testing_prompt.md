### Task Description

Создай V3 artifact package, который содержит ровно один новый project file:

`table-sandbox/v3-pilots/turn_tracker_russian.html`

Это standalone страница для простого хода/раунда в настольной игре или варгейме.

Что должно быть:
- отображение текущего раунда;
- отображение текущей стороны хода;
- кнопка `Следующий ход`;
- кнопка `Предыдущий ход`;
- кнопка `Следующий раунд`;
- кнопка `Сброс`;
- русский интерфейс;
- inline CSS;
- inline JavaScript;
- никаких внешних зависимостей.

Минимальная логика:
- старт: `Раунд 1`, `Ход: Сторона А`;
- `Следующий ход` переключает сторону `А <-> Б`;
- `Предыдущий ход` переключает сторону в обратную сторону;
- `Следующий раунд` увеличивает номер раунда и возвращает ход к `Сторона А`;
- `Сброс` возвращает стартовое состояние.

Визуально:
- аккуратный standalone layout;
- нормальная читаемость;
- без тяжёлого дизайна;
- без фальшивого 3D;
- должен выглядеть нормально на десктопе и на узком окне.

### Required POST_IMPORT_TEST_PROMPT.md

Ты обязан создать `POST_IMPORT_TEST_PROMPT.md` в корне ZIP.

Он должен содержать два явных раздела:
- `Machine checks`
- `Human checks`

Machine checks должны описывать, что удобно проверить обычным Kilo code run, например:
- файл существует по ожидаемому пути;
- это один standalone HTML;
- нет внешних script/link зависимостей;
- есть русские UI-метки;
- есть логика для хода, раунда и сброса;
- если какая-то проверка недоступна, это нужно писать честно.

Human checks должны описывать, что удобно проверить человеку руками, например:
- открыть HTML;
- нажать основные кнопки;
- увидеть смену хода;
- увидеть рост раунда;
- увидеть работу сброса;
- проверить, что интерфейс визуально удобен.

Не пиши, что `Machine checks` или `Human checks` уже были выполнены. Ты создаёшь только prompt, а не test report.
